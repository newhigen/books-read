## 1. 데이터 문해력: 분석보다 활용

입력(생각=사람) → 결과(작업=AI) → 스토리(생각=사람)
* 입력과 스토리의 사고 방식 개선 = '데이터 문해력'

데이터 중심 사고에서 <hi>목적 중심 사고</hi>
* 데이터 중심 사고는 수단에 불과.
    * (wrong) 적당히 건드리다 보면 괜찮은게 뭔가 나오겠지.
    * **데이터에는 해답 따위가 존재하지 않는다.**
* 데이터 분석 자체가 목적이 되기 십상이다.「공감」

레벨 1이 그래프, 2가 데이터, 3이 목적이라고 할 때 1 → 2 → 3 으로 끌어올리지 않고 바로 3에서 시작한다.

## 2. 올바른 데이터로 올바른 문제? - '목적 사고력': 목적에 맞게 문제와 데이터 연결

### 데이터 활용 실패 2가지 이유

2/3의 원인은 같은 곳에. '분석 방법이 틀렸다'가 아니라
1. 풀고자 하는 문제가 명확하지 않다.
2. 정의한 문제와 사용하는 데이터가 일치하지 않는다.

△ 데이터를 가공한다. → 데이터에서 무엇을 알 수 있을지 생각한다.<br/>
O 데이터에서 무엇을 알 수 있을지 생각한다. → 데이터를 가공한다.

### 1. 풀고자 하는 문제가 불명확

<hi>"그래서, 결국 무슨 말이 하고 싶은거지? 그게 본질적이고 가장 중요한 사항인가?<br/>(그래프에서 무엇을 발견했는지는 알겠는데...)"</hi>

목적과 문제를 명확히 하는 것에서 시작[^1]
1. 말하고자 하는게 구체적인가?
    * 목적 -(영향)→ 데이터 → 결과 -(모호)→ 스토리
    * 현상만 보는게 아니다. "그래서 문제가 얼마나 심각한데?"
2. '목적'에 대한 '문제', '원인', '해결방안'을 구분하고 있는가?
    * 원인 자체가 문제가 아니다. e.g.,
      * (원인) 비효율적 영업 (문제) 수주 수 급감
      * (해결방안) 비효율적 영업 (원인) 고객 대응 부실 (문제) 수주 수 급감
    * 가설은 세우는 것은 좋지만, 편견이 들어갈 수 있기 때문에 나중에 하기<br/>「하지만 『완벽한 선택은 없다』에서는 가설을 세우고 행동하라고 한다.이 내용과 결합해보자면 Weak/Flexible 가설을 세우고 빠르게 실행하되 Strong/Fixed 가설은 마지막에 한다. 더 나은 가설을 위해서는 정보가 더 필요한데 실행하면서 더 모이기 때문이다. 『완벽한 선택은 없다』에서 해상도를 높이는 방법.」[^3]
3. 말과 표현의 구체성이 중요하다.
    * 정의에 따라 사용할 데이터와 조사할 사항이 바뀐다.

<hi>나는 문제를 <b>명확하게 정의</b>해 풀고 있는가? Are you solving the <b>right</b> problem?</hi>

### 2. 정의한 문제와 데이터가 불일치

정의한 목적, 문제 -(불일치)- 지표

사례
1. 억지로 데이터 -(연결)- 결론 ⇒ 데이터 중심
2. 주관적으로 이어버림<sup>그렇겠지</sup>[^2]
3. (1) 그래프를 전제로 결론 (2) 과정에 **주관적 해석** 개입[^4]

'지표와 결론의 일치'에 대해 생각할 시간을 가져야한다.

[^1]: 문제를 잘 정의해한다. _ HSL
[^2]: [책] 안다는 착각
[^3]: [책] 완벽한 선택은 없다
[^4]: 지표의 중요성 _ HSL
